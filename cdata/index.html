Do you need CDATA in templates?

February 20, 2018

In my opinion: no.

But it depends.

What does CDATA do?

You may have found CDATA declarations in template files (filetype .phtml),

CDATA marks the text inside as not being XML or HTML. It is ignored by the XML parser.

https://www.w3.org/TR/REC-xml/#sec-cdata-sect

https://magento.stackexchange.com/questions/164569/put-cdata-it-is-a-good-practice-or-not-necessarily-magento-1-9

It’s for validating XHTML markup when JavaScript is inline in the template. It means the validator doesn’t fail and ignores anything between those tags.

This shouldn’t matter if you’re using the latest html5 doctype.

https://stackoverflow.com/questions/66837/when-is-a-cdata-section-necessary-within-a-script-tag?rq=1
